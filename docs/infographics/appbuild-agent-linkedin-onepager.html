app.build Agent • Environment Scaffolding

How app.build turns prompts into production‑ready apps

Scaffolding + validation loops + stack‑aware tooling = reliable prompt‑to‑app generation

1) Orchestrated Scaffolding Pipeline FSM‑guided stages

Stages follow a schema → API → UI finite‑state workflow with stack‑specific templates and validators.

2) Generate → Validate → Repair Loop

  • Context minimized per step (only relevant files + constraints).
  • Sandbox resets between iterations to keep runs deterministic.
  • Artifacts accepted only after validators pass.

3) Validation Stack (per stage)

AB‑01 Boot healthcheck (server boots, UI loads)
AB‑02 Prompt correspondence (no “template‑only” apps)
AB‑03‑06 Create, view/edit, UX sweep, performance checks

4) Scoring & Viability Rubric

Template detection is critical: boot success alone overstates real functionality.

5) Stack‑Aware Tooling

TypeScript / tRPC

  • ESLint + TypeScript type checks
  • Handler unit tests
  • Playwright smoke tests (optional)

PHP / Laravel

  • PHPStan static analysis
  • Laravel feature tests
  • DB migrations + seeders

Python / NiceGUI

  • Ruff + Pyright
  • pytest unit tests
  • Smoke/boot checks
Validators are run per stage inside isolated sandboxes with provisioned Postgres.

6) What the evidence shows (300 experiments + production)

300 End‑to‑end generation runs with automated validators
30 Human‑reviewed apps using the AB rubric
3,000+ Real user apps generated in production (4 months)
73.3% Viable apps with full scaffolding (AB‑01 + AB‑02)
Template trap Boot‑only checks inflate success without AB‑02
Validation tradeoffs Unit tests = quality, E2E tests = brittle
Scaffolded flow Validation gates Repair loop
